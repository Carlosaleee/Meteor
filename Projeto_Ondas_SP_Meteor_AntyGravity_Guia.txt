PROJETO ONDAS SP - ESPECIFICAÇÃO PARA AGENTE LLM (ANTYGRAVITY)

Objetivo

Construir uma aplicação web de previsão do tempo e ondas para Ilha
Comprida/SP.

Regras Gerais

-   Linguagem: Java 21
-   Framework: Spring Boot 3
-   Arquitetura: REST + Clean Architecture
-   Containerização: Docker
-   Banco: PostgreSQL
-   Cache: Redis
-   Frontend: React + Vite + TypeScript + Tailwind
-   Documentação: OpenAPI
-   Testes: JUnit 5

APIs

1.  Open-Meteo (weather + marine)
2.  Stormglass (ondas)
3.  CPTEC/INPE (futuro)

Estrutura Backend

domain/ application/ infrastructure/ api/

Estrutura Frontend

src/ components/ pages/ services/ hooks/ store/ types/

Etapas

Sprint 1

1.  Criar repositório.
2.  Configurar Docker Compose (api, postgres, redis).
3.  Criar projeto Spring Boot.
4.  Configurar Flyway.
5.  Configurar Swagger.
6.  Criar endpoint /health.

Sprint 2

1.  Integrar Open-Meteo.
2.  Criar WeatherService.
3.  Criar Scheduler para sincronização.
4.  Persistir dados.

Sprint 3

1.  Integrar Stormglass.
2.  Criar MarineService.
3.  Persistir ondas.
4.  Criar cache Redis.

Sprint 4

Criar REST: GET /api/weather/current GET /api/weather/hourly GET
/api/waves/current GET /api/waves/hourly GET /api/spots

Sprint 5

Frontend: Dashboard responsivo. Cards. Gráficos. Mapa. Dark mode.

Sprint 6

IA: Receber dados meteorológicos. Gerar resumo em linguagem natural.
Calcular score do surf. Sugerir melhor horário.

Prompt para Agente LLM

Sempre implemente código limpo. Nunca quebre compatibilidade. Escreva
testes. Documente classes. Prefira SOLID. Evite duplicação. Use DTOs.
Use validação. Utilize logs estruturados. Explique decisões
arquiteturais quando necessário.

Critérios

Projeto compilando. Docker funcional. APIs documentadas. Cobertura
mínima de testes: 80%.
